# Release QA Checklist

운영 또는 스테이징 배포 직전에 사용하는 회귀 QA 체크리스트이다.

새 기능 개발을 멈춘 뒤 이 문서와 [local-test-checklist.md](local-test-checklist.md), [operations-readiness.md](operations-readiness.md)를 함께 확인한다.

## 1. 자동 검증

- [ ] Backend 테스트가 통과한다.

```powershell
cd backend
.\gradlew.bat test --no-daemon
```

- [ ] Frontend 빌드가 통과한다.

```powershell
cd frontend
npm run build
```

## 2. 회원/인증

- [ ] 회원가입 이메일 인증이 동작한다.
- [ ] 로그인과 로그아웃이 동작한다.
- [ ] 잘못된 JWT 또는 만료 JWT에서 `/login?reason=expired`로 이동한다.
- [ ] 비밀번호 재설정 이메일 인증이 동작한다.
- [ ] 기존 비밀번호로 로그인이 실패하고 새 비밀번호로 로그인이 성공한다.

## 3. 마이페이지

- [ ] Header의 마이페이지 링크가 `/mypage`로 이동한다.
- [ ] `/mypage` 홈에서 계정, 주문 요약, 기본 배송지, 최근 본 상품이 표시된다.
- [ ] `/mypage/orders`에서 주문 목록과 주문 상세 이동이 동작한다.
- [ ] `/mypage/login-info`에서 비밀번호 변경 모달이 동작한다.
- [ ] `/mypage/profile`에서 이미지, 이름, 소개 변경 모달이 동작한다.
- [ ] `/mypage/addresses`에서 주소 추가, 수정, 삭제, 기본 배송지 설정이 동작한다.
- [ ] 다음 우편번호 검색이 주소 추가/수정 모달에서 동작한다.
- [ ] `/orders`와 `/my/addresses` 기존 경로가 새 마이페이지 경로로 이동한다.

## 4. 상품/장바구니

- [ ] 상품 목록이 표시된다.
- [ ] 상품 검색, 카테고리, 시리즈, 상태, 재고 필터가 동작한다.
- [ ] 상품 상세가 표시된다.
- [ ] 최근 본 상품이 저장되고 마이페이지에 표시된다.
- [ ] `ON_SALE` 상품만 장바구니에 담을 수 있다.
- [ ] 재고보다 큰 수량은 거부된다.
- [ ] 장바구니 수량 변경, 삭제, 비우기가 동작한다.

## 5. 주문/결제

- [ ] 저장 배송지로 주문을 생성할 수 있다.
- [ ] 직접 입력 배송지로 주문을 생성할 수 있다.
- [ ] 주문 생성 후 주문 상태는 `PAYMENT_PENDING`이다.
- [ ] 주문 생성 후 재고 이력 `RESERVED`가 저장된다.
- [ ] Toss 결제창이 정상 호출된다.
- [ ] 결제 성공 후 주문 상태는 `PAID`이다.
- [ ] 결제 성공 후 결제 상태는 `APPROVED`이다.
- [ ] 결제 성공 후 재고 이력 `CONFIRMED`가 저장된다.
- [ ] 결제 성공 페이지 새로고침 시 중복 승인 처리가 안전하다.
- [ ] 결제 실패 또는 결제 대기 취소 시 주문 상태는 `FAILED`이다.
- [ ] 결제 실패 또는 결제 대기 취소 시 재고 이력 `RELEASED`가 저장된다.

## 6. 관리자

- [ ] 일반 회원은 `/admin`에 접근할 수 없다.
- [ ] 관리자 계정은 `/admin`에 접근할 수 있다.
- [ ] 관리자 상품 등록이 동작한다.
- [ ] 관리자 상품 수정이 동작한다.
- [ ] 상품 숨김 처리 후 일반 상품 목록에서 보이지 않는다.
- [ ] 관리자 상품 목록에서는 숨김 상품도 표시된다.
- [ ] 이미지 업로드가 동작한다.
- [ ] 잘못된 이미지 확장자, MIME, 빈 파일, 5MB 초과 파일이 거부된다.
- [ ] 관리자 주문 상태/회원 이메일/기간 필터가 동작한다.
- [ ] `PAID -> PREPARING -> SHIPPED -> DELIVERED` 배송 상태 변경이 동작한다.
- [ ] 관리자 환불이 동작한다.
- [ ] 관리자 환불 반복 요청 시 재고 복구가 중복 처리되지 않는다.
- [ ] 상품/주문/환불 작업이 감사 로그에 남는다.

## 7. 이메일

- [ ] 회원가입 인증 이메일 또는 `MAIL_MODE=LOG` 로그가 남는다.
- [ ] 비밀번호 재설정 인증 이메일 또는 `MAIL_MODE=LOG` 로그가 남는다.
- [ ] 주문 생성 알림이 남는다.
- [ ] 결제 완료 알림이 남는다.
- [ ] 배송 시작 알림이 남는다.
- [ ] 배송 완료 알림이 남는다.
- [ ] 이메일 발송 실패가 주문/결제 트랜잭션을 깨지 않는다.

## 8. 운영 환경

- [ ] `CORS_ALLOWED_ORIGINS`가 실제 프론트 도메인만 허용한다.
- [ ] 운영 Secret이 repo에 없다.
- [ ] 운영 Toss Secret Key는 서버에만 있다.
- [ ] 운영 Toss Client Key는 프론트 공개 키만 사용한다.
- [ ] S3 bucket 권한이 최소화되어 있다.
- [ ] SMTP 발신 도메인 SPF/DKIM/DMARC 설정이 완료되어 있다.
- [ ] health check 또는 장애 알림 채널이 준비되어 있다.
- [ ] 관리자 계정 생성/권한 변경 절차가 정해져 있다.

## 9. 출시 보류 기준

아래 중 하나라도 발생하면 출시를 보류한다.

- 결제 성공 후 주문/결제/재고 상태가 불일치한다.
- 결제 실패 또는 취소 후 재고가 복구되지 않는다.
- 중복 승인 또는 중복 환불에서 재고 이력이 중복 저장된다.
- 일반 회원이 관리자 API 또는 관리자 페이지에 접근할 수 있다.
- 실제 Secret이 repo, 문서, 빌드 결과물에 노출된다.
- 운영 DB migration이 실패한다.
- 운영 프론트가 운영 백엔드가 아닌 다른 API를 호출한다.
